# Brian Chesky: 对话与即兴思考

> 调研维度：深度采访、播客、访谈中的思维过程
> 调研时间：2026年4月

## 一手来源：关键对话记录

### 1. Masters of Scale - Reid Hoffman对谈 (2017, 2021)
**来源**: Masters of Scale podcast，两期节目
**URL**: https://mastersofscale.com/brian-chesky/
**可信度**: 高（一手录音访谈）

**关键即兴片段**：

**关于早期融资的类比（2017期）**：
- Reid Hoffman问Chesky如何说服投资者相信"陌生人住房"这件事
- Chesky的即兴回答："你必须设计一个叙事，让不可能变得不可避免"
- 他讲了一个麦片盒的故事：2008年为了付租金，他和Gebbia设计了Obama O's和Cap'n McCain's麦片盒，卖了800盒，赚了3万美元
- "我们不是在做麦片生意，我们是在证明我们可以用设计创造价值——哪怕是麦片盒这种最不可能的东西"
- Hoffman追问：「但投资者还是say no了，你怎么处理的？」→ Chesky：「被拒绝不是'不'，是'还不行'。你需要找出还缺什么。」

**关于Airbnb"不只是住宿"的即兴定义（2021期）**：
- Chesky当被问"Airbnb是什么公司"时，即兴回答：
- "我们卖的不是床。我们卖的是那种——你走进一个陌生人的家，她给你一把钥匙，说'这就是你的家了'——那种'归属感'。"
- 这个回答后来成为Airbnb品牌的经典表述

### 2. How I Built This - Guy Raz采访 (2017)
**来源**: NPR How I Built This podcast
**URL**: https://www.npr.org/2017/04/17/524271800/brian-chesky-airbnb
**可信度**: 高

**关键即兴片段**：

**关于"疯狂想法"的类比**：
Guy Raz问Chesky怎么看"让人们把家租给陌生人"这个想法为什么别人觉得疯狂？
- Chesky："如果你说'我有一个可以装1万首歌的设备'，这在iPod出现之前听起来也很疯狂。"
- "好的想法在一开始都看起来像坏主意。如果不是这样，早就有人做了。"
- 这句话后来成为创业圈频繁引用的名言

**关于被拒绝的韧性**：
Chesky详细描述了被15位天使投资人拒绝的经历：
- "我们给15个人发了Airbnb的deck，8个人没回，5个人回了说'我们不投'，2个人说'这是个灾难'"
- "但有一个投资人Fred Wilson——他当时说'我不理解为什么有人会做这个'——后来这成了他最公开承认错过的投资之一"
- Chesky的结论不是"他们错了"，而是"我们没表达对"

**改立场瞬间**：
- 一开始Chesky很坚持"Airbnb不做酒店"，但在Guy Raz追问下承认："我们早期对'专业房东'有过偏见。后来我意识到，如果你在Airbnb上有1000个房源，你的标准应该比有1个房源的人更高，但不代表你做错了什么。"

### 3. Sam Altman访谈 - "How to Build the Future" (2017)
**来源**: Y Combinator, YouTube系列
**URL**: https://www.youtube.com/watch?v=pb_zmSrb4vM
**可信度**: 高

**核心洞见即兴表达**：

**关于产品设计哲学**：
- Altman问：「Airbnb是怎么做到比竞争对手更好的住宿体验的？」
- Chesky：「我们不是从'比对手好10%'开始的。我们是问'让我们设计一个你在世界上任何地方都能感到归属的体验。'如果你从那里开始，10%不是一个足够大的目标。」
- "真正好的产品不是在做feature comparison。你在创造一个新的类别（category）。"

**关于"设计师创业"的即兴辩护**：
- Chesky当被问及"设计师创业者有什么不同？"时：
- "设计师被训练的不是解决问题，而是发现问题。工程师想'我怎么能更快到达X'，设计师先问'X是不是正确的目的地？'"
- "但两者不是对立的，它们是一起的。最好的设计师懂工程逻辑，最好的工程师有设计直觉。我在Airbnb聘请了很多比我更懂技术的人，但我是那个问'我们到底要做什么'的人。"

### 4. The Verge - Decoder (2023)
**来源**: The Verge, Decoder podcast with Nilay Patel
**URL**: https://www.theverge.com/2023/11/6/23949679/airbnb-brian-chesky-ai-design-verification-updates
**可信度**: 高

**关于AI的即兴思考**：
- Patel问：AI会不会让Airbnb过时？
- Chesky的即兴回应："AI不会取代旅行。没有人会说'ChatGPT，让我体验一次旅行。'AI帮你简化旅程中所有烦人的部分——搜索、比较、预订——让你有更多时间做旅行中真正有意义的事：探索、连接、体验。"
- 这个框架（AI消除摩擦→留下核心体验）体现了Chesky一贯的"设计优先减少摩擦"思维

**关于"创始人模式"的即兴回应**：
- "我在疫情后撤销了很多我之前的决定。我曾经被教育说'做一个好的CEO就是雇好的人然后让他们自己干'。我发现对于Airbnb这种设计驱动的公司，这是错误的。"
- "我必须亲自看产品。我必须自己是用户。我每年住50-100晚Airbnb。"

### 5. GGV Fellows interview - 对话Hans Tung (2018)
**来源**: GGV Capital, YouTube
**URL**: https://www.youtube.com/watch?v=来自GGV频道
**可信度**: 高

**关于规模化的即兴比喻**：
- "做一个产品和服务1亿人不是1万人的线性放大。它是一个完全不同的操作。就像做一顿饭和开一个餐厅。"
- "当你有1亿用户时，你的问题从'这个功能好用吗？'变成'这个功能在1000种不同的场景中会出什么问题？'"
- Chesky在这里展示了他对于"规模"的理解不是量的增长，而是质的变化

### 6. Stanford GSB - View From The Top (2023)
**来源**: Stanford Graduate School of Business
**URL**: 无需具体URL，这是公开演讲
**可信度**: 高

**关于领导力演变的坦诚**：
- 被学生问："你怎么看待自己的变化？"
- Chesky："我刚做CEO时，我以为我的工作是说'是'或'不'。后来我发现我的工作是问问题，不是回答问题。再后来我发现——"
- "我的工作是建立共识。不是自上而下告诉所有人做什么，而是让所有人对目标有一致的理解，然后他们自己知道做什么。"
- 这是一个罕见的Chesky承认自己"早期领导模式错误"的时刻

## 被追问时的高频反应模式

1. **用类比解释复杂概念**：Chesky几乎在被问到每个问题上都会给出一个类比
   - 产品设计 → Walt Disney的故事板
   - 公司文化 → "像做一顿饭vs开餐厅"
   - 创业 → "iPod出现之前没人想随身带1万首歌"
   - 信任 → "Airbnb是陌生人之间的婚姻——短暂但有亲密"

2. **从"我们当时不知道"开始**：在复盘早期决策时，他习惯先承认当时的无知，再解释现在理解了什么。这是一种知识上的谦卑姿态，但同时也让他的叙述更具权威感（因为"现在我懂了"）。

3. **被问困难问题时，会先停顿然后说"That's a great question"**：这不是礼貌套路，而是他组织思考的时间。通常停顿时他在寻找一个类比。

4. **立场演变的叙事模式**：他描述改变立场的方式是"I used to think X... but then I learned Y"。这比"我之前错了"更微妙——不是承认错误，而是强调成长。

## 少见的冲突/防御时刻

有几个场景Chesky明显变得防御性：

1. **2019-2020年的监管问题**：当被追问Airbnb对住房市场的负面影响时，Chesky倾向于把问题重新框定为"我们需要更好的监管框架"而非"我们的商业模式是否可持续"。

2. **被问及与专业房东的关系**：早期他坚持Airbnb是"业余分享"。后来他承认这个定义过于狭窄，但调整姿态的过程很慢。

3. **关于IPO定价和盈利问题**：在被问及"Airbnb什么时候真正盈利"时更倾向于谈"长期价值"而非短期财务。
